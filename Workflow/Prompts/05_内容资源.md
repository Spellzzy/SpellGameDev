# 阶段5：内容与资源 Prompt 模板

## 5.1 数值设计

```
我需要设计【系统名称】的数值体系。

类型：【成长曲线/战斗公式/经济系统/掉落概率/...】
参考：【有没有参考的游戏或风格偏好】
关键参数：
- 【参数1：含义和大致范围】
- 【参数2：含义和大致范围】

请帮我：
1. 给出核心公式
2. 用表格列出各等级/阶段的数值
3. 画出成长曲线的趋势（文字描述）
4. 标注需要策划调参的变量
5. 给出对应的 C# 数据结构和配置文件模板
```

## 5.2 PCG 算法设计

```
我需要一个程序化生成【地图/关卡/地形/物品/...】的算法。

要求：
- 生成规模：【大致尺寸/数量】
- 风格约束：【规则描述】
- 性能约束：【生成时间要求】
- 可控性：【哪些参数需要开放给策划调】

请先给出伪代码方案，确认思路后再写实现。
```

## 5.3 资源规格定义

```
我需要为【模块名称】定义资源规格。

资源类型：【UI图标/角色模型/场景贴图/特效/音频/...】
目标平台：PC + 手机

请给出：
1. 推荐的格式和尺寸
2. 命名规范
3. 目录组织方式
4. 压缩/图集设置建议
5. 内存预算估算
```

## 5.4 场景 UI 结构生成

```
我需要为【游戏/模块名称】生成完整的场景 UI 结构。

场景包含的面板：
- 【面板1名称】：【包含的 UI 元素列表】
- 【面板2名称】：【包含的 UI 元素列表】
- 【面板N名称】：【包含的 UI 元素列表】

已有的 View 脚本：
- 【View1.cs — 职责描述】
- 【View2.cs — 职责描述】

要求：
1. 编写 Editor 脚本通过 [MenuItem] 一键生成
2. 自动创建 Canvas + EventSystem + Camera
3. 按 View 脚本中的 [SerializeField] 字段自动创建对应 UI 元素
4. 通过 SerializedObject 绑定所有引用，无需手动拖拽
5. 预制体（如 Cell、Item 等）自动生成并保存到 Assets/Prefabs/
6. 面板使用 Layout 组件自动排版
7. 弹窗类面板默认隐藏

请先分析所有 View 脚本的 SerializeField，再设计 Hierarchy 结构，最后编写生成代码。
```

## 5.5 批量资源生成

```
我需要批量生成【资源类型】。

具体需求：
- 数量/规模：【N 个/张/组】
- 规格：【尺寸、格式、命名规则】
- 变体规则：【不同变体之间的差异逻辑】

请帮我编写 Editor 脚本：
1. 通过 MenuItem 或 EditorWindow 提供入口
2. 程序化创建资源（Texture2D / Sprite / Material / Prefab 等）
3. 按命名规范保存到指定目录
4. 输出生成日志
```

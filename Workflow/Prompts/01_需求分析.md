# 阶段1：需求分析 Prompt 模板

## 1.1 新功能需求分析

```
我要做一个【功能名称】系统。

背景：【为什么要做这个功能，解决什么问题】
核心玩法：【一句话描述核心交互】
参考：【类似的游戏/系统，如果有】

请帮我：
1. 分析需求边界（哪些要做、哪些不做）
2. 梳理核心数据结构
3. 设计 public API 接口
4. 画出状态流转（文字版）
5. 列出和现有系统的依赖关系
6. 标注风险点和边界条件
7. 预估工作量（小/中/大）
```

## 1.2 策划文档转技术方案

```
以下是策划文档内容：
---
【粘贴策划文档】
---

请帮我：
1. 提取核心功能点，列成清单
2. 标注每个功能点的技术可行性（✅容易 / ⚠️需要注意 / ❌有风险）
3. 给出技术实现方案
4. 如果有不明确或矛盾的地方，列出来让我去和策划确认
```

## 1.3 需求变更评估

```
原功能：【当前功能描述】
变更需求：【策划要改什么】

请评估：
1. 影响范围（哪些文件/模块需要改）
2. 改动量级（小/中/大）
3. 是否需要重构架构
4. 推荐的实施步骤
5. 可能的风险
```

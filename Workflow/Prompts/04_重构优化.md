# 阶段4：重构与优化 Prompt 模板

## 4.1 架构重构

```
当前问题：【描述架构上的痛点】
例如：XxxManager 越来越臃肿 / A和B模块耦合太深 / 新需求加不进去了

相关代码：
---
【粘贴代码或说明文件位置让AI读取】
---

请帮我：
1. 诊断当前设计的核心问题
2. 给出2-3个重构方案（各有优劣）
3. 推荐一个方案并说明理由
4. 给出分步实施计划（每步可验证）
```

## 4.2 性能优化

```
需要优化的场景：【描述性能要求】
当前表现：【FPS/加载时间/内存占用】
目标表现：【期望的指标】

请帮我制定优化方案：
1. 排查清单（按可能性排序）
2. 每项优化的预估收益
3. 实施优先级
4. 是否有副作用
```

## 4.3 需求变更适配

```
原来的设计：【描述原来怎么做的】
现在策划要改成：【描述新需求】
矛盾点：【改动和现有架构冲突在哪】

请帮我：
1. 评估能否在现有架构上改（还是需要重构）
2. 给出最小改动方案
3. 给出理想方案（如果有时间的话怎么做更好）
4. 标注改动影响的文件清单
```

# 阶段3：测试与调试 Prompt 模板

## 3.1 Bug 排查

```
遇到一个Bug：

现象：【描述你看到的表现】
复现步骤：
1. 【步骤1】
2. 【步骤2】
3. 【步骤3】

预期行为：【应该怎样】
实际行为：【实际怎样】

相关代码/日志：
---
【粘贴报错日志或相关代码】
---

请帮我分析可能的原因，并给出修复方案。
```

## 3.2 性能问题排查

```
性能问题：

现象：【卡顿/帧率低/加载慢/内存飙升】
出现时机：【什么操作后出现】
Profiler数据（如果有）：【截图或关键数值】

相关代码：
---
【粘贴可能有问题的代码段】
---

请帮我：
1. 分析性能瓶颈
2. 给出优化方案（按收益排序）
3. 标注每个优化的预期效果
```

## 3.3 生成测试用例

```
请为以下类/方法生成 NUnit 测试用例：

---
【粘贴要测试的代码】
---

要求：
- 覆盖正常路径、边界条件、异常输入
- 使用 [TestCase] 参数化测试
- 如果依赖Unity运行时，标注为 PlayMode 测试
- 每个测试方法名用 方法名_条件_期望 格式
```

## 3.4 代码审查

```
请审查以下代码：

---
【粘贴代码】
---

审查要点：
- [ ] 命名规范
- [ ] 内存泄漏风险（事件没取消订阅、协程没停止等）
- [ ] 空引用风险
- [ ] 性能隐患（GC、频繁查找等）
- [ ] 线程安全
- [ ] 可扩展性
- [ ] 边界条件处理

按严重程度分类输出（🔴严重 🟡建议 🟢良好）。
```

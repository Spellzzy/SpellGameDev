# 阶段2：编码实现 Prompt 模板

## 2.1 新建功能模块

```
请帮我实现【模块名称】模块。

功能需求：
- 【需求点1】
- 【需求点2】
- 【需求点3】

技术约束：
- 遵循 Manager + Data + View 三层架构
- 通过 EventSystem 和其他模块通信
- 配置数据走 JSON/ScriptableObject

请按以下顺序输出：
1. 数据层（Data类）
2. 逻辑层（Manager类）
3. 表现层（View/UI类）
4. 事件定义（如需新增事件）
5. 使用示例
```

```
我需要在【已有模块名】上增加【新功能描述】。

当前代码：
---
【粘贴相关代码，或让AI自己读文件】

```

## 2.5 Editor 场景搭建脚本

```
请帮我编写一个 Editor 脚本，一键生成【游戏/模块名称】的完整场景结构。

UI 结构需求：
- 【面板1：包含哪些元素】
- 【面板2：包含哪些元素】
- 【面板3：包含哪些元素】

技术要求：
- 通过 [MenuItem] 暴露菜单入口
- 使用 SerializedObject 自动绑定所有 [SerializeField] 引用
- 需要生成的预制体自动保存到 Assets/Prefabs/ 对应目录
- Canvas 使用 ScreenSpace-Overlay + CanvasScaler（ScaleWithScreenSize）
- 布局使用 GridLayoutGroup / VerticalLayoutGroup / HorizontalLayoutGroup
- 弹窗面板初始 SetActive(false)
- 包含 EventSystem + Camera + 游戏入口对象

参考已有实现：
- Scripts/Editor/SudokuSceneBuilder.cs（数独场景搭建脚本）

请按以下顺序实现：
1. 分析所有 View 脚本的 [SerializeField] 字段和 UI 层级需求
2. 创建预制体生成方法（含 SerializeField 绑定）
3. 创建各面板构建方法
4. 创建绑定方法（通过 SerializedObject 绑定所有引用）
5. 主入口方法串联所有步骤
```

## 2.6 Editor 预制体生成脚本

```
请帮我编写一个 Editor 脚本，自动生成【预制体名称】预制体。

预制体结构：
- 根节点组件：【Image + Button + XxxView】
- 子节点列表：
  - 【子节点1：Text，用途说明】
  - 【子节点2：Image，用途说明】
  - 【子容器：GridLayoutGroup，包含 N 个子元素】

要求：
- 通过 [MenuItem] 暴露菜单
- 创建完成后通过 PrefabUtility.SaveAsPrefabAsset 保存到指定路径
- 使用 SerializedObject 绑定组件上的所有 [SerializeField] 字段
- 销毁场景中的临时对象，只保留预制体文件
```

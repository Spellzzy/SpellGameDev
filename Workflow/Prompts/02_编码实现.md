# 阶段2：编码实现 Prompt 模板

## 2.1 新建功能模块

```
请帮我实现【模块名称】模块。

功能需求：
- 【需求点1】
- 【需求点2】
- 【需求点3】

技术约束：
- 遵循 Manager + Data + View 三层架构
- 通过 EventSystem 和其他模块通信
- 配置数据走 JSON/ScriptableObject

请按以下顺序输出：
1. 数据层（Data类）
2. 逻辑层（Manager类）
3. 表现层（View/UI类）
4. 事件定义（如需新增事件）
5. 使用示例
```

## 2.2 在已有模块上扩展功能

```
我需要在【已有模块名】上增加【新功能描述】。

当前代码：
---
【粘贴相关代码，或让AI自己读文件】
---

要求：
- 不改已有的 public API（如需改请说明原因）
- 新增的逻辑用扩展方式加入，不侵入原有流程
- 说明改了什么、为什么这样改
```

## 2.3 重复代码提取模板

```
以下几个地方有类似的代码逻辑：
- 【文件A 的 xx方法】
- 【文件B 的 xx方法】
- 【文件C 的 xx方法】

请帮我：
1. 分析共同点和差异点
2. 提取公共基类或工具方法
3. 重构后各处改为调用公共代码
4. 保证行为不变
```
